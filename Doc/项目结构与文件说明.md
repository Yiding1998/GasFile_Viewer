## 1. 文档目标

本文面向第一次接触仓库的用户和维护者，用较短篇幅说明项目结构、主要文件用途、维护责任和常见修改入口。需要了解算法、数据结构和关键函数时，请阅读[代码设计与实现说明](代码设计与实现说明.md)；需要学习工作台操作时，请阅读[中文使用手册](Garfield_gas_workbench_pro_使用说明.md)。

文档适用于 2026-07-15 的项目结构，索引格式为 schema v3。

## 2. 项目概览

GasFile_Viewer 是一个无需后端服务器的静态网页项目，包含三部分：

1. Garfield++ `.gas` 文件集合及其内容索引。
2. 气体文件检索页面和中英文气体文件工作台。
3. 索引生成、测试和 GitHub Pages 自动部署工具。

在线数据流为：

```text
GasFile 中的气体文件
        -> Python 索引生成器
        -> GasFile/gas_index.json
        -> 独立检索页或工作台检索面板
        -> 选择气体文件并在工作台解析、绘图和导出
```

| 文件 | 类型 | 功能 | 维护建议 |

| `index.html` | 手工维护 | GitHub Pages 根入口，根据浏览器语言转到中文或英文在线工作台。 | 保持轻量，不在此复制工作台功能。 |
| `gas_file_search.html` | 手工维护 | 独立英文检索页。加载索引，提供成分、比例、温度、压强、质量和文本检索，支持排序、共享链接、CSV 导出及跳转工作台。 | 修改查询界面或结果展示时编辑。匹配算法应优先修改共享核心。 |
| `assets/js/gas-search-core.js` | 手工维护 | 独立检索页和工作台检索面板共用的单位换算、匹配、评分及排序逻辑。 | 任何检索规则修改都应增加测试，并确认两个入口行为一致。 |

| `assets/js/workbench-library.js` | 手工维护 | 向中英文工作台注入“搜索仓库气体文件”面板，负责读取索引、安全下载、哈希校验、去重和批量加入工作台。 | 修改仓库检索集成、安全限制或批量下载时编辑。 |
| `garfield_gas_workbench_pro.html` | 手工维护 | 中文工作台。包含本地化解析提示、文件管理、比较绘图、分析、拟合、热图、导出和项目保存等功能。 | 与英文版本保持功能和控件 ID 同步。 |
| `garfield_gas_workbench_pro_english.html` | 手工维护 | 英文工作台，功能应与中文工作台一致。 | 功能修改需同步到中文版本，只有界面文字不同。 |
| `garfield_workbench_offline_zh.html` | 自动生成 | 中文单文件离线工作台，内嵌全部共享 JavaScript，可直接读取本地气体文件。 | 不要直接编辑；运行单文件生成器更新。 |
| `garfield_workbench_offline_en.html` | 自动生成 | 英文单文件离线工作台。 | 不要直接编辑；运行单文件生成器更新。 |
| `garfield_gas_multi_file_viewer_advanced_legend.html` | 兼容跳转 | 保留旧公开 URL，并跳转到 `legacy/` 中的实际页面。 | 不要删除，以免历史链接失效。 |
| `legacy/garfield_gas_multi_file_viewer_advanced_legend.html` | 兼容保留 | 较早的中文多文件查看器，功能独立于当前工作台。 | 仅修复严重问题；新功能加入 `pro` 工作台。 |
| `README.md` | 手工维护 | 英文项目首页、在线入口、用户流程和新增气体文件说明。 | 用户可见功能变化时同步更新。 |

## 5. `GasFile/` 数据目录

该目录保存实际 Garfield 气体文件，当前按 Ar、Ne、RPC 典型混合气和单一纯气体等类别组织。文件名和目录名只用于辅助识别；索引优先读取文件内部 `Identifier:`、`Dimension:`、电场、磁场和夹角等内容。

| 文件或内容 | 类型 | 功能 | 是否手动修改 |
|---|---|---|---|
| 各类气体文件 | 数据 | Garfield++ 生成的输运参数表，是检索和工作台绘图的数据源。 | 可以新增或替换，但应重新生成索引并运行测试。 |
| `gas_aliases.json` | 配置 | 将 `argon`、`R134a`、`isobutane` 等别名归一化到标准气体名称。 | 新增同义名称时修改。 |
| `gas_metadata_override.json` | 配置 | 按仓库相对路径覆盖无法自动正确解析的文件元数据。默认为空对象。 | 仅用于确认过的特殊文件，不应用来掩盖普遍解析错误。 |
| `gas_index.json` | 自动生成 | schema v3 检索索引，包含成分、比例、标准化温压、质量标记、哈希和输运网格覆盖范围。 | 不建议手动编辑；运行生成器更新。 |
| `gas_index_report.md` | 自动生成 | 汇总解析状态、质量警告、成分统计以及需要人工复核的文件。 | 不建议手动编辑。 |

气体文件数量较多，因此文档不逐个列出。每个气体文件在索引中都有一条独立记录，可通过路径和 SHA-256 追踪。

## 6. 工具、测试和自动化

| 文件 | 类型 | 功能 |
|---|---|---|
| `tools/build_gas_index.py` | 手工维护 | 扫描 `GasFile/`，解析文件内容和路径回退信息，应用别名与人工覆盖，生成索引和质量报告，并检查索引是否过期。 |
| `tools/build_standalone_workbenches.py` | 手工维护 | 将共享解析器、检索核心和工作台集成库嵌入中英文工作台，生成两个只需下载一个 HTML 的离线版本。 |
| `tests/test_build_gas_index.py` | 手工维护 | 测试温压单位、Identifier 解析、质量标记、输运网格覆盖和索引生命周期。 |
| `tests/test-gas-file-parser.js` | 手工维护 | 使用共享模块解析仓库中的全部气体文件，并锁定普通记录和扩展记录的对齐行为。 |
| `tests/test_web_integration.py` | 手工维护 | 检查中英文工作台、共享脚本、安全限制、参数化横坐标和双语工作台链接等静态集成契约。 |
| `.github/workflows/gas-search.yml` | 手工维护 | 在推送、拉取请求或手动触发时运行测试、重建并检查索引，再将整个仓库部署到 GitHub Pages。 |
| `.gitignore` | 手工维护 | 忽略 Python 缓存和编译产物，避免无关文件进入版本库。 |

## 9. 自动生成文件与维护边界

- `GasFile/gas_index.json` 和 `GasFile/gas_index_report.md` 由生成器维护。
- GitHub Pages 部署时会在工作流临时环境中重建索引，但不会创建机器人提交。
- 本地克隆需要执行生成器才能让已提交索引反映新气体文件。
- PDF 不由当前 GitHub Actions 自动生成，Markdown 手册改变后应确认 PDF 是否也需要更新。
- 中英文 `pro` 工作台仍有平行的界面和绘图代码，但 Garfield 解析和物理换算统一由 `assets/js/gas-file-parser.js` 提供。

## 10. 最小维护检查

```bash
python3 tools/build_gas_index.py --pretty
python3 tools/build_gas_index.py --check
python3 -m unittest discover -s tests -v
node tests/test-gas-file-parser.js
git diff --check
```

提交前还应通过本地 HTTP 服务器打开中英文工作台和独立检索页，确认浏览器能够读取 `GasFile/gas_index.json`。
