# 1. Document Overview

This manual applies to the latest single-file web application:

```text
garfield_gas_workbench_pro_english.html
```

[Open the English workbench online](https://yiding1998.github.io/GasFile_Viewer/garfield_gas_workbench_pro_english.html).

The application reads ASCII `.gas` files generated by Garfield, Garfield++, or Magboltz directly in a web browser. It supports multi-file comparison, physical-unit conversion, uncertainty visualization, interpolation, normalization, scans, fitting, two-dimensional plots, multi-panel figures, and publication-quality export.

No installation of Garfield++, ROOT, Python, or a web server is required. Gas files are read locally by the browser and are not automatically uploaded to the Internet.

## 1.1 Typical Applications

- Compare drift velocity, diffusion, Townsend coefficient, and attachment coefficient for different gas mixtures.
- Compare transport parameters at different temperatures, pressures, magnetic fields, or field grids.
- Study the effective Townsend coefficient $\alpha_{\mathrm{eff}}=\alpha-\eta$ and locate its zero crossings.
- Scan a transport parameter as a function of gas fraction, pressure, or temperature.
- Produce figures and data files for papers, group meetings, presentations, and analysis reports.
- Save a complete analysis project and continue editing it later.

## 1.2 Recommended Browsers

Use a recent version of Google Chrome, Microsoft Edge, or Mozilla Firefox. A desktop browser with a screen width of at least 1280 px is recommended.

For local use, download and open only `garfield_workbench_offline_en.html`. This single-file version embeds every JavaScript dependency and can read local `.gas` files. If the browser restricts local storage, plotting templates may not be saved permanently. The **Save Full Project** function can still be used in that case.

# 2. Quick Start

## 2.1 Minimal Workflow

1. Open the [English workbench](https://yiding1998.github.io/GasFile_Viewer/garfield_gas_workbench_pro_english.html), or download and open the single-file offline version `garfield_workbench_offline_en.html`.
2. Drag one or more `.gas` files into the upload area, or click **Add Files**.
3. In **Main Plot Settings - Data and Comparison**, select the X variable and the left Y parameter.
4. In the loaded-file list, enable the curves to display and adjust their order, color, line style, and marker style.
5. Use the mouse wheel, pan, or box zoom in the interactive plot.
6. In **Figure Size and Export**, choose a size preset and export PNG, SVG, or PDF.

## 2.2 Recommended Multi-Gas Comparison Workflow

Assume that several files correspond to increasing SF6 fraction:

1. Load all files at once.
2. Select **Specified Gas Fraction** as the automatic sort key.
3. Choose SF6 as the gas component and select ascending order.
4. Click **Sort**.
5. In the legend settings, enable **Gas Composition** and disable **File Name**.
6. Apply a colorblind-friendly palette and uniform line width and marker size.
7. Select `E/N` for the X axis and `alphaEff = alpha - eta` for the Y axis.
8. Enable logarithmic X or Y scaling when physically appropriate.
9. Export SVG or PNG using the single-column or double-column paper preset.

## 2.3 Search and Add Files from the Repository

When the workbench is opened through GitHub Pages or a local repository web server:

1. Select **Search repository gas files** near the initial file picker or beside **Add Files**.
2. Enter gas components and optional fractions, temperature, pressure, quality, alias, or path filters.
3. Review each candidate's composition, conditions, file size, Garfield version, GASOK availability, E/p range, magnetic-field range, angle range, and grid dimensions.
4. Select individual files or select the closest three or five results.
5. Select **Add selected files**. Successfully parsed files are appended to the normal workbench file list.

The loader accepts only index-listed paths below `GasFile/`, limits total selected size and download concurrency, supports cancellation, and verifies SHA-256 when Web Crypto is available. Duplicate repository paths or hashes are skipped. Repository path, SHA-256, index generation time, and indexed metadata are included in saved project files.

The standalone gas-file search page also has **English workbench** and **中文工作台** actions. These open the selected result in the corresponding localized workbench and add it after index validation.

Opening the HTML through `file://` keeps all original local loading, plotting, analysis, export, and project features. Repository search requires HTTP through GitHub Pages or a local web server.

# 3. Supported Files and Parsing Rules

## 3.1 Supported File Type

The program is intended primarily for ASCII `.gas` files from Garfield, Garfield++, or Magboltz. A valid file should normally contain:

- `Version`
- `GASOK bits`
- `Identifier`
- `Dimension`
- `E fields`
- `E-B angles`
- `B fields`
- `Mixture`
- `The gas tables follow:`
- `PGAS` and `TGAS` in the footer

If the gas-table marker, footer, or grid description cannot be found, the application reports a parsing error.

## 3.2 Multidimensional Grids

The application can read gas tables containing:

- Electric-field grids
- Magnetic-field grids
- Electric-field-to-magnetic-field angle grids
- Excitation channels
- Ionization channels

The main plot displays one magnetic-field and one angle slice at a time. When multiple files have different grids, the nearest available magnetic-field and angle slice is selected independently for each file.

## 3.3 Pressure and Temperature

The footer is used to read:

- Pressure `PGAS`, treated internally in Torr
- Temperature `TGAS`, in kelvin

The gas number density is calculated using the ideal-gas relation:

$$
N=\frac{p}{k_{\mathrm B}T}.
$$

Pressure is converted to pascals before this calculation.

# 6. Main Plot Settings: Data and Comparison

## 6.1 X-Axis Variables

The X-axis selector contains:

- Actual electric field `E [V/cm]`
- Reduced electric field `E/p [V/(cm*Torr)]`
- Reduced electric field `E/N [Td]`
- Every built-in transport parameter available to the left and right Y axes
- User-defined derived parameters
- Individual excitation and ionization channels available in the active file

The conversions are:

$$
E=(E/p)p,
$$

$$
E/N=\frac{E}{N}.
$$

`E/N` is displayed in townsend, where `1 Td = 10^{-21} V*m^2`.

Selecting a transport parameter for X creates a parametric plot of one transport quantity against another. Points follow the original electric-field sweep by default, preserving curve structure when the selected X parameter is non-monotonic. The order can be changed to ascending X, or the connection mode can be set to points only.

The same parameter may be selected for X and Y, which normally produces `y=x`. Records with an invalid X or Y value are skipped, and the main-plot note reports the skipped count.

## 6.2 Magnetic-Field and Angle Slices

- **E-B Angle Slice** selects the target field angle.
- **Magnetic Field B Slice** selects the target magnetic field.

A file with only one angle or magnetic field has only one option. If files use different grids, the application selects the nearest available slice. It does not automatically interpolate in magnetic field or angle.

## 6.3 Left and Right Y Axes

The left Y axis is the primary parameter. A second parameter may be assigned to the right Y axis to create a dual-axis plot.

Recommendations:

- Use dual axes only when the two quantities have a clear reason to be compared.
- Give both axes explicit names and units.
- Avoid arbitrary axis scaling that could create a misleading visual correlation.

## 6.4 Comparison Modes

The following modes are available.

### Original Values

Display the data from each file without reference-based transformation.

### Difference from Reference

$$
\Delta y_i(x)=y_i(x)-y_{\mathrm{ref}}(x).
$$

### Percentage Difference from Reference

$$
\Delta y_i^{\%}(x)=
\frac{y_i(x)-y_{\mathrm{ref}}(x)}{y_{\mathrm{ref}}(x)}\times100\%.
$$

If the reference value is zero, that point may be undefined.

### Ratio to Reference

$$
R_i(x)=\frac{y_i(x)}{y_{\mathrm{ref}}(x)}.
$$

### Normalize Each Curve to Its Own Maximum

$$
\widetilde y_i(x)=\frac{y_i(x)}{\max_x y_i(x)}.
$$

### Normalize Each Curve at a Specified X Value

$$
\widetilde y_i(x)=\frac{y_i(x)}{y_i(x_0)}.
$$

Enter `x0` in **Normalization X Value**. For a field-coordinate X axis, the current interpolation method is used when `x0` is not a native grid point. For a transport-parameter X axis, the nearest native parameter point is used to avoid non-unique inversion of a non-monotonic parameter.

## 6.5 Reference File

Difference, percentage, and ratio modes require a reference file. With a field-coordinate X axis, the reference file should cover the compared X range. With a transport-parameter X axis, files are aligned on the underlying `E/p` coordinate before reference differences, percentages, or ratios are calculated.

## 6.6 Common X Grid

Available choices are:

- **Original Grid for Each File** - preserve each curve's own grid.
- **Reference-File Grid** - map all curves to the reference file's X grid.
- **Union of All File Grids** - use the union of all X values.
- **Common Original Points Only** - keep only points present in every file.
- **Custom Uniform Grid** - enter start, end, and step values.

For ratio, difference, and consistent point-by-point comparison, the reference-file grid or a custom grid is usually recommended.

When a transport parameter is used for X, the reference, union, common, and custom grids all refer to the underlying `E/p` grid. The application processes both X and Y parameters on that grid before constructing the parametric curve. Custom grid start, end, and step values therefore use the `E/p` coordinate.

## 6.7 Interpolation Methods

### Linear Interpolation

Interpolate linearly between neighboring data points.

### Logarithmic-X Interpolation

Interpolate in logarithmic coordinate space. With a transport-parameter X axis, interpolation uses the underlying positive `E/p` coordinate rather than directly interpolating a potentially non-monotonic transport parameter.

### No Interpolation - Common Original Points Only

Use only original points with matching X values. This option is appropriate for strict quantitative comparisons where interpolation must be avoided.

Tooltips identify interpolated points. When using interpolation in a paper or formal analysis, document the chosen method.

## 6.8 Curve Connection Modes

Supported modes:

- Straight-line connection
- Original points only
- Step plot
- Smoothed visual curve

**Parametric Point Order** provides:

- **Original Electric-Field Sweep** - the default; preserves the physical order as the field increases.
- **Ascending X Value** - useful when the intended presentation explicitly requires X sorting.
- **Native Points Only** - selected through the curve-connection control and draws no connecting line.

The smoothed curve is for presentation only. It is not a new Garfield++ calculation. Statistics, exported data, and quantitative conclusions should preferentially use original points or explicitly generated interpolation points.

## 6.9 File Uncertainties

If the `.gas` format contains uncertainty information, the program can display:

- Y error bars
- Y error bands
- Y error bars and bands together
- Horizontal error bars for a transport parameter selected as X

Y uncertainties are shown only in **Original Values** mode. Horizontal bars are unavailable for field-coordinate X axes or parameters without uncertainty data. Some derived parameters and gas-table versions do not contain uncertainty information.

## 6.10 Point Labels and Crosshair Readout

- **Show Point Labels** displays selected numeric labels rather than labeling every point when curves are dense.
- **Crosshair and Synchronized Readout** displays values from multiple curves near the cursor position.
- Parametric X axes use the nearest plotted point rather than attempting inverse interpolation of a non-monotonic parameter.
- Tooltips and synchronized readout also show the source `E`, `E/p`, magnetic field, and angle.

# 11. Supported Physical Quantities and Conversions

## 11.1 Drift Velocity

- `vE [cm/ns] = raw.ve * 10^-3`
- `vE [cm/us] = raw.ve`
- `vB_perp [cm/ns] = raw.vb * 10^-3`
- `vExB [cm/ns] = raw.vexb * 10^-3`

## 11.2 Diffusion

Let pressure `p` be expressed in Torr:

$$
D_L=\frac{D_{L,\mathrm{raw}}}{\sqrt p},\qquad
D_T=\frac{D_{T,\mathrm{raw}}}{\sqrt p}.
$$

The application provides:

- `DL [sqrt(cm)]` and `DT [sqrt(cm)]`
- `DL [um/sqrt(cm)]` and `DT [um/sqrt(cm)]`

## 11.3 Townsend and Attachment Coefficients

The stored values are restored from logarithmic form:

$$
\frac{\alpha}{p}=\exp(\mathrm{raw.alpha}),
$$

$$
\frac{\eta}{p}=\exp(\mathrm{raw.eta}).
$$

Therefore:

$$
\alpha=p\exp(\mathrm{raw.alpha}),
$$

$$
\eta=p\exp(\mathrm{raw.eta}).
$$

The effective Townsend coefficient is calculated by the web application:

$$
\alpha_{\mathrm{eff}}=\alpha-\eta.
$$

The corresponding reduced coefficient is:

$$
\frac{\alpha_{\mathrm{eff}}}{p}=
\frac{\alpha}{p}-\frac{\eta}{p}.
$$

## 11.4 Number-Density-Reduced Quantities

The application provides:

$$
\frac{\alpha}{N},\qquad
\frac{\eta}{N},\qquad
\frac{\alpha-\eta}{N}.
$$

Number density is converted from `m^-3` to `cm^-3` for these calculations.

## 11.5 Other Derived Quantities

- Electron mobility:
  $$
  \mu_e=\frac{v_E}{E}.
  $$
- `DL/vE`
- `DT/vE`
- Ion mobility
- Lorentz angle
- Ion dissociation coefficient
- Six diffusion-tensor components
- Total excitation and ionization rates
- Individual excitation- and ionization-channel rates

The Lorentz angle and some rate quantities retain the units used in the source file. Confirm the source and units in **Raw Data and Parameter Sources** before quantitative use.

# 12. Analysis: Feature Points and Statistics

## 12.1 Statistical Interval

An X minimum and maximum can be entered for the statistical interval. Leave both fields blank to use the full curve range.

## 12.2 Statistical Summary

For each visible curve, the program calculates:

- Number of points
- Minimum value and its X coordinate
- Maximum value and its X coordinate
- Mean
- Median
- Trapezoidal integral
- Zero crossings or sign changes

The numerical integral is:

$$
\int y\,dx\approx
\sum_i\frac{y_i+y_{i+1}}{2}(x_{i+1}-x_i).
$$

## 12.3 Feature Points

The program identifies:

- Minimum
- Maximum
- Zero crossings or sign changes
- An approximate plateau start for drift-velocity curves

A zero crossing is estimated by linear interpolation between neighboring points with opposite signs. Plateau detection is a screening heuristic and should not replace a dedicated physical criterion.

Create an effective coefficient normalized by number density:



## 13.3 Safety Restrictions

Expressions are evaluated locally. The application blocks semicolons, code blocks, arrow functions, and objects such as `window`, `document`, `fetch`, `eval`, `Function`, and `localStorage`. Even so, use only mathematical expressions that you understand and trust.

# 14. Analysis: Curve Fitting

## 14.1 Fit Settings

Select:

- File
- Parameter
- Fit model
- Polynomial order
- X minimum and maximum

## 14.2 Polynomial Fit

The model is:

$$
y=\sum_{k=0}^{n}c_kx^k.
$$

Coefficients are solved using normal equations. High polynomial orders or very large X ranges may be numerically unstable. Avoid high-order fits without a clear need.

## 14.3 Exponential Fit

The model is:

$$
y=A\exp(Bx).
$$

Only points with `y > 0` are used. Parameters are obtained from a linear fit to `ln y`.

## 14.4 Fit Results

The application reports:

- Fit expression
- Coefficient of determination `R^2`
- Number of points used

The fitted curve is overlaid on the main plot as a separate dashed line. It is a web-calculated result, not original Garfield++ data.

# 20. Plotting Templates versus Full Projects

## 20.1 Plotting Templates

A template mainly stores plotting settings such as:

- Axes
- Titles
- Legend
- Font sizes
- Comparison and interpolation settings

Templates are stored in browser `localStorage` and apply only to the current browser profile. Clearing browser data or using private browsing may remove them.

## 20.2 Full Projects

A full project stores the original gas files and all application state. It can be copied to another computer and reopened there. For long-term research, saving the full project is strongly recommended.

# 21. Recommended Research Workflows

## 21.1 Publication-Quality Multi-Gas Comparison

1. Load all files.
2. Inspect the parsing report.

4. Apply a colorblind-friendly palette.

1. Select `alphaEff = alpha - eta`.
2. Use a linear Y axis.
3. Inspect zero crossings in **Feature Points and Statistics**.
4. Add a horizontal guide at `Y = 0`.
5. Zoom into the crossing region.

1. Select `E/N` or `E/p` as the X axis.
2. Use the reference-file grid.
3. Select a suitable interpolation method.
4. Use ratio or percentage mode.
5. Choose the baseline-pressure file as the reference.

# 22. Troubleshooting

## 22.1 File Cannot Be Read

Possible causes:

- The file is not an ASCII `.gas` file.
- `The gas tables follow:` is missing.
- The `H Extr:` footer is missing.
- `Dimension` does not match the number of data values.
- The file is truncated or damaged.

Read the status message and parsing report, then inspect the file header and footer in a text editor.

## 22.2 A Parameter Is Missing from the List

Possible causes:

- The GASOK flag marks the parameter as unavailable.
- The file has no corresponding channel.
- Channel definitions differ between the current file and other files.

Switch the current file and inspect the parameter list again.

## 22.3 No Data Are Visible

Check whether:

- All files are hidden.
- A logarithmic axis filtered zero or negative values.
- Manual minimum and maximum values were entered in reverse order.
- The selected magnetic-field or angle slice contains data.
- The reference file covers the required range in comparison mode.
- A custom grid lies outside the data range.
- No common points exist in no-interpolation mode.

## 22.4 Ratio or Percentage Values Are Unstable

If the reference value is close to zero, ratios and percentage differences can become extremely large or undefined. Inspect the reference curve and restrict the analysis interval.

## 22.5 Error Bars Are Not Shown

- The file may not contain uncertainty information.
- The selected parameter may not have implemented uncertainty conversion.
- The comparison mode may not be **Original Values**.
- Uncertainty values may be zero or non-finite.

## 22.6 Legend Composition Is Unexpected

Composition display prefers the Identifier and normalizes `iC4H10` and `i-C4H10`. The Mixture array is used only when the Identifier has no parseable fractions. If fallback output is still unexpected, verify the original header, Mixture values, and version-specific gas-number mapping.

## 22.7 Templates Cannot Be Saved

The browser may block local storage. Possible solutions:

- Exit private-browsing mode.
- Allow local storage for the page.
- Use **Save Full Project** instead.

## 22.8 PDF Text or Layout Is Unsatisfactory

Try one of the following:

- Export SVG and convert it to PDF with Inkscape, Illustrator, or PowerPoint.
- Use the browser's Print or Save as PDF function.
- Increase export size and font sizes.
- Avoid placing a very long legend inside the plot.

# 23. Limitations and Scientific Cautions

1. The application parses, converts, and visualizes data; it does not replace Garfield++ or Magboltz calculations.
2. Interpolation, fitting, smoothing, and feature-point detection are secondary processing results and must be distinguished from original data.
3. Magnetic-field and angle slices use nearest-neighbor selection, with no automatic interpolation in those dimensions.
4. Units for Lorentz angle, excitation rates, ionization rates, and some tensor quantities depend on the source-file definition. Verify them before quantitative use.
5. Number-density conversion assumes that pressure and temperature metadata are correct and that the ideal-gas approximation is appropriate.
6. High-order polynomial fits can be numerically unstable. A high `R^2` does not imply physical validity.
7. Dual-Y-axis plots can be visually misleading. Label both axes and units clearly.
8. Custom expressions should contain only trusted mathematical formulas.
9. A project file may contain complete original gas data. Check for sensitive or unpublished information before sharing it.
10. For reproducibility, archive the original `.gas` files, project file, exported CSV, and plotting settings before publication.
